查看网络 IP 和 网关
	网络 IP 的查询：	
		widows：cmd 中输入 ipconfig 查询
		Linux：终端输入 ifconfig 查询
	相关的链接：可以通过 ping 操作进行 虚拟机 和 主机 之间的相互连接
	VmWare提供的三种网络连接模式：
		桥接模式:
			概念：虚拟机可以直接连接外部物理网络的模式，主机起到了网桥的作用。这种模式下，虚拟机可以直接访问外部网络，并且对外部网络是可见的。
			工作原理：Internet->路由->Pc ->网桥->交换机->VM虚拟机
			特点：架构简单，路由及以后的设备都在一个局域网中，连接同意个路由之后可以相互进行访虚拟机
			缺点：没有隐私，并且子网最后一位只有 0 - 255 ，共有254个IP可供使用
		NAT模式(Network Adress Translation)：
			概念：虚拟机和主机构建一个专用网络，并通过虚拟网络地址转换(NTA)设备对 IP 地址进行转换。
			         虚拟机通过共享主机 IP 可以访问外部网络，但外部网络无法通信。
			工作原理：Internet ->路由->PC(其中虚拟了一个 VMnet8 网卡）->路由->VM虚拟机
				这个 VMnet8网卡 使得 主机 与 虚拟机 再同一个局域网中 
		仅主机模式：
			虚拟机只于 主机共享一个专用网络，与外部网络无法通信。
	修改静态IP：
		原因：因为默认的 虚拟机 IP 是动态的不方便进行连接访问，所以修改成静态 IP
		修改步骤：
			首先再 Linux 虚拟机中 终端中输入：vim /etc/sysconfig/network-scripts/ifcfg-ens33
			修改第四行 BOOTPROTO = "static"
			然后复制以下代码：(需要注意的是，修改的时候 网关应当和 VmWare 中的 网关地址一样,IP地址：只能修最后的三位)
				#IP 地址
				IPADDR=192.168.11.100
				#网关
				GATEWAY=192.168.11.2
				#域名解析器
				DNS1=192.168.11.2

系统管理：
	计算机中，一个正在执行的程序或命令，被叫做“进程”（process）。
	启动之后一只存在、常驻内存的进程，一般被称作“服务”（service）。
	管理service：
	CentOs6:
		基本语法：service 服务名称 start | stop | restart | status
		查看服务的方法：ls /etc/init.d/
		例子：（1）查看网络服务的状态：service network status
	CentOs7:
		基本语法：systemctl start | stop | restart | status 服务名
		查看服务的方法：ls /usr/lib/systemd/system
		例子：查看防火墙服务的状态：systemctl status firewalld

	设置后台自启配置：
		相关的运行级别内容可以查看 课件 6.6
		CentOs6:
			chkconfig --list (功能描述：查看服务器开机启动状态，但是是按照等级来进行划分的)
			chkconfig network off（功能描述：关闭所有的 network服务的 等级)
			chkconfig network on （功能描述：打开所有的 network服务的 等级)
			chkconfig network --level 3 off（功能描述：关闭第三等级）
		CentOs7相关指令：
			systemctl list-unit-files （功能描述：查看服务开机启动状态）
			systemctl disable service_name （功能描述：关掉指定服务的自动启动）
			systemctl enable service_name （功能描述：开启指定服务的自动启动）
			setup（可以直接打开系统服务，设置对应的开机自启动)
		关机重启命令：

				（1）sync （功能描述：将数据由内存同步到硬盘中）

	文件权限：
		33.文件属性(有关权限问题)：
			（1）通过 ls -l 或者 ll 来查看文件属性

			（2）相关说明：Linux系统是一种典型的多用户系统，不同的用户处于不同的地位，拥有不同的权限。为了保护系统的安全性，Linux系统对不同的用户访问同一文件（包括目录文件）的权限做了不同的规定。在Linux中我们可以使用ll或者ls -l命令来显示一个文件的属性以及文件所属的用户和组。

			（3）从左到右边十个字符表示
				文件类型(1)         	属主权限(3)	属组权限(3)	其他用户权限(3)
				       0		 1  2  3		   1  2  3		      1  2  3
				 d(目录文件)           r(读)w(写)x(执行)       r(读)w(写)x(执行)	r(读)w(写)x(执行)
				
				对于上面的十个字符表示的相关解释：
					1）在Linux中第一个字符代表这个文件是目录、文件或链接文件等等
					- 代表文件；
					d 代表目录；
					l 链接文档(link file)；
					2）第1-3位确定属主（该文件的所有者）拥有该文件的权限。---User
					3）第4-6位确定属组（所有者的同组用户）拥有该文件的权限，---Group
					4）第7-9位确定其他用户拥有该文件的权限 ---Other
					5）其中相关权限的意思为：
					 r(red)：; w(write); x(excute); 
				例子：-rw-r--r--. 1 root root 1891 4月   8 19:13 initial-setup-ks.cfg
					这里除了十个字符串还多出了一点东西
					1）. 1：连接数，如果是文件代表的是硬链接的个数，需要提及的是硬链接为零的时候次为文件词可以删除。
					2）.1：链接数，如果是文件夹代表的是子文件夹的个数	例如：drwx------. 15 iron   iron 4096 4月  11 13:47 iron
					3）1891：文件大小
					4）4月   8 19:13：最近修改时间
					5）initial-setup-ks.cfg：文件名称

			（4）chmod 改变权限
				第一种方式变更权限
					chmod [{ugoa}{+-=}{rwx}] 文件或目录
					u:所有者 g:所有组 o:其他人 a:所有人(u、g、o 的总和)
					例子：chmod u-x,o+x houge.txt
				第二种方式变更权限(此处是将 r w x 当作三个二进制进行组合)
					chmod [mode=421 ] [文件或目录]
					r=4 w=2 x=1 rwx=4+2+1=7
				选项：-R：递归修改文件夹里面的所有权限
					例子：chmod -R 777 xiyou/
						
			(5)chown 改变所有者(改变属主)
				chown [选项] [最终用户] [文件或目录] （功能描述：改变文件或者目录的所有者）
					选项：-R：递归操作

			
			(6)chgrp 改变所属组(改变数组)
				chgrp [最终用户组] [文件或目录] （功能描述：改变文件或者目录的所属组）

	（1）RPM（包管理工具)
		基本理念：RPM（RedHat Package Manager），RedHat软件包管理工具，类似windows里面的setup.exe是Linux这系列操作系统里面的打包安装工具，它虽然是RedHat的标志，但理念是通用的。

		基本语法：
			查询：rpm -qa （功能描述：查询所安装的所有 rpm 软件包）
			卸载：rpm -e RPM软件包 （功能描述：卸载软件包）
		          	          rpm -e --nodeps 软件包（卸载软件时，不检查依赖。这样的话，那些使用该软件包的软件在此之后可能就不能正常工作了。）
			安装：rpm -ivh RPM包全名
		         		选项说明：
					-i：install，安装
					-v：--verbose，显示详细信息
					-h：--hash，进度条
					--nodeps：安装前不检查依赖
	
	（2）YUM 仓库配置
		YUM概述：YUM（全称为YellowdogUpdater,Modified）是一个在Fedora和RedHat以及CentOS 中的Shell前端软件包管理器。基于RPM包管理，能够从指定的服务器自动下载RPM包并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包，无须繁琐地一次次下载、安装。
		基本用法：yum[选项][参数]
		选项：-y：队友有的提问都回答：“yes"
		参数说明：
			install 安装rpm软件包
			update 更新rpm软件包
			check-update  检查是否有可用的更新rpm软件包
			remove 删除指定的rpm软件包
			list 显示软件包信息
			clean 清理yum过期的缓存
			deplist 显示yum软件包的所有依赖关系	
		修改YUM的镜像网站改变：查看word8.2.3，但是没有修改的必要，会自动进行寻找到最近的国内镜像网站，所以下载的速度还是比较快的。
